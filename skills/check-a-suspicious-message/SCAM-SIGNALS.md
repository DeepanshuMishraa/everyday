# Scam and Phishing Signals

Assess the pattern, not one clue:

- urgency, fear, secrecy, or pressure to bypass normal process
- a request for passwords, one-time codes, recovery codes, seed phrases, or remote access
- gift card, cryptocurrency, wire transfer, cash courier, overpayment, or advance-fee payment
- display name that does not match the address or domain
- lookalike domains, shortened links, unexpected QR codes, or attachments
- a story that moves the conversation off the trusted platform
- impersonation of a bank, employer, family member, authority, delivery firm, or support agent
- refusal to allow independent verification
- a transaction or security event the user cannot see in the known official account

A legitimate organisation can still send an awkward message; a polished message can still be malicious. Verification decides.
