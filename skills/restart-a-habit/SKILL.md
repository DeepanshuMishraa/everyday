# Restart a Habit

Treat the break as evidence about the design, not the user's character.

## Decide whether the habit still deserves to exist

Ask what outcome the habit served, what the old routine was, when it broke, what changed, and whether the outcome still matters. Choose **keep, redesign, or retire**. Retiring a habit that no longer serves the user is a valid result.

## Diagnose the break

Use [HABIT-BREAKS.md](HABIT-BREAKS.md). Identify one primary failure mode and supporting evidence. Do not diagnose ADHD, depression, addiction, eating disorders, compulsions, or other clinical conditions.

## Build the restart

Define:

- a stable cue the user already encounters
- a minimum action specific enough to observe
- materials or environment prepared in advance
- an optional expansion that is never required
- a recovery action after a miss
- a review date after a short trial

The minimum must still represent the real behaviour; “think about exercising” is not an exercise habit.

## Recovery rule

After a miss, perform the minimum at the next normal cue. Do not make up repetitions, impose punishment, or reset a public streak. If repeated misses share one friction, redesign that friction before adding reminders.

Return the diagnosis, restart protocol, miss rule, and review criteria.
